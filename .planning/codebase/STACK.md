## Installation & Distribution

**Entry Points:**
- `bin/install.js` - Interactive/non-interactive installer with runtime and location selection
- Bin reference: `get-shit-done-cc` command (points to `bin/install.js`)

**Distributed Files (see `package.json` files field):**
- `bin/` - Installation and CLI entry points
- `commands/` - GSD command definitions (30+ markdown files)
- `get-shit-done/` - Core GSD system (bin, templates, references, workflows)
- `agents/` - 11 agent definitions for specialized workflow tasks
- `hooks/dist/` - Pre-built hook scripts (generated by `npm run build:hooks`)
- `scripts/` - Build and utility scripts

## Code Organization

**Main Entry Points:**
- `bin/install.js` - Installation wizard (`~1800 lines`)
- `get-shit-done/bin/gsd-tools.cjs` - CLI tool for project operations (`~170kb compiled`)
- `get-shit-done/bin/gsd-tools.test.cjs` - Test suite (`~85kb`)

**Hooks (ran by Claude Code/OpenCode/Gemini):**
- `hooks/gsd-check-update.js` - Background update checker
- `hooks/gsd-statusline.js` - Terminal statusline display

**Command Format:**
- Commands are markdown files with YAML frontmatter defining allowed tools
- Example: `commands/gsd/help.md`, `commands/gsd/plan-phase.md`

## Security Model

**No External Dependencies:**
- Zero npm dependencies minimizes supply chain risk
- All functionality built from Node.js stdlib

**Configuration Safety:**
- Environment variables supported:
  - `BRAVE_API_KEY` - Optional web search (handled in `gsd-tools.cjs`)
  - `CLAUDE_CONFIG_DIR`, `OPENCODE_CONFIG_DIR`, `GEMINI_CONFIG_DIR` - Config overrides
- Secrets stored in user home directory, not committed to git
- `.gitignore` excludes: `.claude/`, `.DS_Store`, `node_modules/`

**File Manifest Tracking:**
- `gsd-file-manifest.json` - SHA256 hashes of installed files
- `gsd-local-patches/` - User modifications backed up on update
- Enables safe updates without losing local customizations
